박스 태그

div 태그

div는 Division의 약자로, 웹페이지의 섹션이나 구역을 정의하는 데 사용됩니다.

제목

이 내용들이 하나의 그룹으로 묶였습니다

div 태그의 특징:

  • 다른 HTML 요소들을 그룹화하는 컨테이너 역할
  • 레이아웃 구성에 매우 유용
  • CSS 스타일링을 적용할 때 자주 사용

링크와 이미지

하이퍼링크 (a 태그)

구글로 이동

네이버 (새 탭)

섹션1으로 이동

주요 속성:

  • href: 이동할 링크 주소
  • target="_blank": 새 탭에서 열기

이미지 (img 태그)

주요 속성:

  • src: 이미지 파일 경로
  • alt: 이미지가 표시되지 않을 때 나타날 대체 텍스트
  • width, height: 이미지 크기 설정

주요 태그:

  • <table>: 테이블 전체를 감싸는 태그
  • <tr>: 테이블의 행(row)
  • <th>: 헤더 셀 (굵게 표시됨)
  • <td>: 일반 데이터 셀
  • rowspan: 세로 병합, colspan: 가로 병합

Form 요소

폼은 사용자로부터 입력을 받는 인터랙티브한 부분입니다. 사용자가 입력한 데이터는 서버로 전송되어 처리됩니다.

기본 입력 요소들

주요 속성들

  • name: 서버로 데이터를 보낼 때 식별자 역할 (라디오 버튼은 같은 name으로 그룹화)
  • value: 서버로 전송될 실제 값
  • placeholder: 입력 필드에 표시되는 안내 텍스트
  • required: 필수 입력 필드로 지정
  • checked: 라디오 버튼이나 체크박스를 기본 선택 상태로 설정
  • selected: 옵션을 기본 선택 상태로 설정

레이블 (label)

레이블은 폼 요소에 설명을 붙이는 역할을 합니다:

레이블의 장점:

  • 접근성 향상 (스크린 리더 사용자에게 도움)
  • 레이블을 클릭해도 해당 입력 필드가 선택됨
  • CSS 스타일링에 유용

시맨틱 태그

시맨틱 태그는 HTML5에서 도입된 중요한 개념으로, 웹 페이지의 내용에 의미를 부여하는 태그들입니다.

시맨틱 태그의 장점

  1. 검색 엔진 최적화(SEO): 검색 엔진이 페이지 내용을 더 정확하게 파악
  2. 코드 가독성: 각 부분의 역할이 명확해져 코드 이해가 쉬워짐
  3. 접근성 향상: 스크린 리더 등 보조 기술이 페이지 구조를 더 잘 이해

각 태그의 역할

  • <header>: 페이지나 섹션의 머리말 영역
  • <nav>: 내비게이션 링크들의 집합
  • <main>: 페이지의 주요 콘텐츠 (한 페이지에 하나만 사용)
  • <section>: 문서의 구역을 나누는 태그
  • <article>: 독립적으로 구분되는 콘텐츠
  • <aside>: 주요 내용과 간접적으로 연관된 부분
  • <footer>: 페이지나 섹션의 바닥글 영역
